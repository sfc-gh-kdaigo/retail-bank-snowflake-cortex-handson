# 🏦 リテールバンキング向け Snowflake Intelligence ハンズオン

地方銀行のリテール営業業務をサポートするAIエージェントを構築するSnowflake Cortex AIハンズオン

---

このアセットは、**Snowflake Cortex AI**の主要機能を活用し、リテール営業担当者・渉外担当者向けのAIアシスタントを構築するためのハンズオンです。

## ✨ 主な目的

自然言語でデータ分析・情報検索が可能なAIアシスタントを構築し、その仕組みを理解します。

**具体シナリオ**
*   **個人・法人顧客**の預金取引情報、顧客属性情報を自然言語で分析
*   **名寄せデータ**を活用し、統合顧客単位での取引状況を把握
*   顧客セグメント分析（年齢層、業種、取引規模など）による営業ターゲティング支援
*   取引パターン分析（入出金頻度、残高推移、チャネル利用傾向）による顧客理解
*   Cortex Agent/Snowflake Intelligenceによる統合的な情報提供

## 📚 このハンズオンで学ぶこと

本アセットを通じて、以下のスキルや知識を習得できます。
*   ✅ Snowflake Cortex AI の全体像と主要な機能（Cortex Agent、Analyst、Search、Intelligence）
*   🤖 Cortex Agentによる複数ツール（Analyst/Search/Sproc）のオーケストレーション
*   📊 **Semantic View**の具体的な実装手順（日本語カラム名対応）
*   🔗 **名寄せデータ**を活用した統合顧客分析の実現方法
*   📈 銀行業務KPI（純資金流入額、顧客純増数など）のメトリクス定義
*   🔄 **DATA_AGENT_RUN**によるSQL内でのAgent実行とTask連携

## 🗄️ 利用データ（テーブル構成）

### 対象テーブル一覧

| テーブル名 | 粒度 | 用途 |
|:---|:---|:---|
| **流動性預金取引データ＿日次** | 取引単位 × 日次 | 預金取引の履歴（入出金、残高など） |
| **顧客基本属性情報＿月次** | 顧客単位 × 月次 | 顧客マスタ情報（氏名、住所、業種など） |
| **統合顧客＿インデクス＿月次** | 顧客単位 × 月次 | 名寄せマッピング（統合顧客番号） |

### テーブル間のリレーション

```
流動性預金取引データ＿日次
    │
    │ 顧客番号 + 金融機関コード
    ▼
統合顧客＿インデクス＿月次  ←── 名寄せテーブル
    │
    │ 顧客番号 + 金融機関コード
    ▼
顧客基本属性情報＿月次
```

### 主要カラム説明

#### 顧客基本属性情報＿月次
| カラム名 | 説明 |
|:---|:---|
| 顧客番号 | 全店で一意に識別される顧客番号 |
| 漢字氏名１ | 取引上の名義人名称 |
| 人格コード | 顧客の人格（個人、法人等）を識別 |
| 業種コード | 取引先の業種（日銀業種分類） |
| 資本金 | 法人の資本金（百万円単位） |
| 年収 | 個人顧客の年収（万円単位） |
| 元帳状態表示 | 活動状態（00：活動中、99：未取引先） |
| 取引開始日 | 顧客開設日 |
| 閉鎖日 | 顧客閉鎖日 |

#### 流動性預金取引データ＿日次
| カラム名 | 説明 |
|:---|:---|
| 入払区分 | 入金（1）/ 出金（2）の区分 |
| 取引金額 | 取引金額 |
| 取引後残高 | 取引後の表面残高 |
| チャネル識別コード | ATM / 窓口 / オンライン等のチャネル |
| 外部取引コード | 取引の種類を識別するコード |

#### 統合顧客＿インデクス＿月次
| カラム名 | 説明 |
|:---|:---|
| 統合顧客番号 | 銀行グループ全体で顧客を一元管理する番号 |
| 代表顧客番号 | 同一統合顧客内の代表顧客番号 |

## 📊 ビジネスKPI

### 取引系KPI（流動性預金取引データ＿日次）
| KPI名 | 計算式 |
|:---|:---|
| **純資金流入額** | SUM(入払区分=1の取引金額) - SUM(入払区分=2の取引金額) |
| **総取引金額** | SUM(取引金額) |
| **総取引件数** | COUNT(*) |
| **平均取引金額** | AVG(取引金額) |

### 顧客系KPI（顧客基本属性情報＿月次）
| KPI名 | 計算式 |
|:---|:---|
| **総顧客数** | COUNT(元帳状態表示=0) |
| **新規顧客数** | COUNT(取引開始日が当月) |
| **解約顧客数** | COUNT(閉鎖日が当月) |
| **顧客純増数** | 新規顧客数 - 解約顧客数 |

## 🏗️ アーキテクチャ概要

```
┌─────────────────────────────────────────────────────────────┐
│                  Snowflake Intelligence                     │
│                    （チャットUI）                            │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                      Cortex Agent                           │
│          （オーケストレーション・ツール選択）                  │
│  ┌────────────┬────────────┬────────────────────────────┐  │
│  │   Tool 1   │   Tool 2   │         Tool 3             │  │
│  │  Analyst   │   Search   │      Stored Proc           │  │
│  │            │   (自社)   │                            │  │
│  └────────────┴────────────┴────────────────────────────┘  │
└─────────────────────────────────────────────────────────────┘
       │              │              │
       ▼              ▼              ▼
  Semantic View   内部規定        SEND_EMAIL
  （顧客・取引   商品説明書      等のアクション
    構造化データ） PDF等
```

### コンポーネント詳細

| コンポーネント | 役割 |
|:---|:---|
| **Snowflake Intelligence** | Cortex Agentを利用するためのチャットボットUI。営業担当者が自然言語で質問を入力。 |
| **Cortex Agent** | ユーザーの質問を解析し、適切なツール（Analyst/Search/Sproc）を自動選択・実行するオーケストレーション層。 |
| **Cortex Analyst（ツール）** | Semantic Viewを通じて構造化データ（顧客・取引）に対して自然言語からSQLを生成・実行。 |
| **Cortex Search（ツール）** | 内部規定、商品説明書、マニュアル等に対するRAG検索を提供。 |
| **Stored Procedure（ツール）** | メール送信などのアクション実行。 |

## 💡 環境構築と利用方法

### Step 1: 環境構築・データ投入
1. `01_db_setup.sql` を実行してデータベース・テーブル・サンプルデータを作成
2. 日本語カラム名版と英語カラム名版の両方が作成されます

### Step 2: Semantic View作成
3. `03_sv_setup.sql` および `03_semantic_view_japanese_columns.md` を参考にSnowsight GUIでSemantic Viewを作成

### Step 3: Cortex Search設定（RAG）
4. `04_rag_setup.sql` を実行してCortex Search Serviceを作成
5. `files/` フォルダ内のPDFをステージにアップロード

### Step 4: Cortex Agentの作成
6. `06_agent_design.md` を参考にCortex Agentを作成し、以下のツールを登録：
   - **Cortex Analyst**: 作成したSemantic Viewを指定
   - **Cortex Search**: 作成したSearch Serviceを指定

### Step 5: Snowflake Intelligenceへ公開
7. Agentを公開し、Snowflake Intelligenceから作成したCortex Agentを選択してチャット開始
